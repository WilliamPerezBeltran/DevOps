Tutorial de configuración básica de Jenkins GitScm, GitHub Webhooks y Ngrok

 Prerequisitos:

 Jenkins Server
Tener instalada una instancia de Jenkins server en tu ordenador y los complementos necesarios para este (Java v8 | Java v11). En este tutorial lo utilizaremos con Docker y docker compose.
Acá te dejo la especificación del archivo Dockerfile (para la definición de la imagen) y el docker-compose.yml (para la construcción del servicio y el montado del volumen).
 Dockerfile
 docker-compose.yml
Una vez tengas estos dos archivos en tu ordenador (de preferencia en una ruta exacta), corre el comando ($ docker-compose up -d) para construir la imagen y levantar el servicio (jenkins) el cual expone el server en el puerto 8080 (contenedor) al puerto de nuestra máquina anfitriona (host).
Para comprobar que Jenkins está corriendo de manera exitosa puedes ejecutar los siguientes comandos o acceder a tu navegador (localhost:8080):
Shell
Copy

Los resultados que tenemos en pantalla son la real muestra de que nuestro localhost se encuentra ya en el internet bajo una administración proxy de ngrok.

 Bonus: Definiciones

Jenkins: Es un servidor de automatización Open Source (escrito en Java) y usado para agilizar el proceso de building y deployment de un proyecto de software (desde el más básico hasta el más complejo).
SCM (Source Code Managment): Por su traducción (Gestión de Código Fuente), hace referencia a como un determinado software (Git) es capaz de llevar el control y reacción de cambios en el código fuente de un proyecto. Este concepto normalmente suele estar relacionado a los sistemas de control de versionamiento.
Ngrok: Herramienta de reverse proxy (proxy inverso) que lleva tráfico a una dirección (URL) pública a través de procesos ngrok (connection tunnels) que se encargan de conectarse a una dirección local que especifiquemos.
Docker: Proyecto de código abierto que permite construir, distribuir y ejecutar software a través de contenedores.
GitHub Webhook: Mecanimo de GitHub especializado en notificar a un servidor externo que ha existido un cambio o alteración en el código fuente de un proyecto (repositorio u organización).
Pipeline Job: Un pipeline es una suite o conjunto de funciones que admiten la Integración y Entrega Continua (Continuous Integration & Delivery) para trabajos (jobs) en Jenkins.
